# Demo Sierra Vista Project Instructions

## Journal Maintenance

**CRITICAL**: Whenever a discrete, meaningful action is completed through an AI chat session, you MUST update JOURNAL.md with a single-line entry documenting what was accomplished.

### What constitutes a "meaningful action":

- File creation or deletion
- Code refactoring or significant changes
- Deployment to AWS or other services
- Configuration changes (package.json, vite.config.js, etc.)
- Repository setup or Git operations
- Feature implementation or bug fixes
- Infrastructure changes (S3, CloudFront, etc.)

### What does NOT require a journal entry:

- Simple questions or explanations
- Reading files or searching code
- Small typo fixes or formatting changes
- Exploratory work that doesn't result in changes

### Journal Format:

Follow the existing format in JOURNAL.md:

```
- **Step X**: [Brief description of what was done] (March 30, 2026)
```

Keep entries to a single line for brevity and clarity. Increment the step number based on the last entry.

## Project Context

This is a Vue.js 3 project built with Vite, deployed to AWS S3 and CloudFront.

- **Framework**: Vue.js 3 with Vite
- **Deployment**: AWS S3 (static hosting) + CloudFront CDN
- **Region**: us-east-1
- **Dev Server**: http://localhost:3000
